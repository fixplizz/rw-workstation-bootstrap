# Karpathy Guidelines

This note records the lightweight working style guidelines used for this
workspace.

## Principles

- Make it work first, then make it clear, then make it nicer if needed.
- Prefer simple, readable solutions over clever ones.
- Keep iteration fast and feedback loops short.
- Favor directness in code and documentation.
- When in doubt, choose the smallest change that satisfies the task.
